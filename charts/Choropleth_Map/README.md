# 点密度地图 Dot Density Map

图表分类: 地理 Geospatial  
感知排名: ⭐⭐⭐⭐☆

## 简介

点密度地图用点的数量和密度表示绝对数值分布。
它基于 pyecharts Geo 组件，使用内置中国城市坐标库：

- 每个点代表固定数量单位（如 1 万人、100 个事件）
- 点越密集的区域表示总量越大
- 支持多分组着色，区分不同类别

相比面量图，点密度图更强调**离散个体分布**和**绝对数值**。

## 数据格式

需要包含以下字段：

- label：地区名称（如"北京"、"西陵区"）
- value：绝对数值（Numeric，如总人口、事件数量）
- category（可选）：分组列，不同组用不同颜色区分

单分组示例：

| 城市 | 从业人数(万) |
|------|------------|
| 北京 | 58 |
| 上海 | 52 |
| 广州 | 35 |

多分组示例（按民族）：

| 城市 | 民族 | 人口 |
|------|------|------|
| 北京 | 汉 | 1000 |
| 贵阳 | 苗 | 300 |
| 贵阳 | 侗 | 250 |

## 适用场景

点密度地图适用于以下场景：

- 城市间人口/从业者总数对比
- 连锁门店的地理分布
- 不同民族/群体的城市分布
- 事件频率地理分布（犯罪、投诉等）
- 物流订单的发货地分布

## 典型场景

- 全国主要城市互联网从业者分布（单分组）
- 各省市不同民族人口分布（多分组，按民族着色）
- 连锁餐饮品牌门店全国分布
- 电商退货纠纷按城市分布

## 使用建议

- 数据应为**绝对数值**（总量、计数等），而非相对数值
- 数据量大时设置 `dots_per_unit`（如 100），避免点数过多影响渲染
- 多分组（category）时图例自动显示，颜色按麦肯锡配色循环
- `chart_type` 推荐：
  - `scatter`（静态点）：数据量多，追求清晰
  - `effectScatter`（动态涟漪）：数据量少，追求视觉冲击
- 地名需与 pyecharts 内置库匹配（3700+ 中国城市/区县）

## 注意事项

- 每行数据代表一个地点的总量，通过 value 控制点的数量权重
- dots_per_unit > 1 时，n_points = value / dots_per_unit（向上取整到至少1）
- 地名不匹配时该行跳过并记录 warning
- 数据格式为长格式：每行一个地点/分组组合

## 与相似图表的区别

### 点密度图 vs 面量图

| 对比维度 | 点密度图 | 面量图 |
|---------|---------|--------|
| 视觉编码 | 点的数量/密度 | 区域填充颜色深浅 |
| 数据表达 | 绝对数值（总量） | 相对数值（密度、比率） |
| 常见用途 | 人口总数、犯罪事件数 | 人口密度、GDP per capita |
| 区域面积影响 | ✅ 能反映区域大小差异 | ❌ 不影响（颜色才是关键） |
| 绝对值 vs 相对值 | 绝对值 | 相对值 |
| 视觉可读性 | 密集区点会重叠，需调整透明度 | 颜色对比清晰，易于分类 |
